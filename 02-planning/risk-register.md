# Risk Register

| ID | Risk | Likelihood | Impact | Mitigation | Status |
|----|------|-----------|--------|------------|--------|
| R1 | **Fabricated metrics** — an agent invents a savings %, accuracy, or cost number to make a POC look complete (esp. tempting given papers' big headline numbers) | Medium | Critical | Every number must trace to a captured live call in `04-logs/live-evidence-ledger.md`; paper claims labeled `Research supported, NOT reproduced here`; plan uses "TBD — measured live in <POC>" for unmeasured values; Opus honesty review greps distillation for unsourced numbers; per-section evidence label | Open |
| R2 | **LLM-judge error** — judge mis-scores an open-ended answer, corrupting accuracy | Medium | High | Closed tasks (coding/math/qa) use **deterministic** graders (subprocess unit tests / numeric / normalized match), never the judge; judge used ONLY for ensemble fusion/ranking, not final correctness; judge prompt isolates instructions in system, constrains output, deterministic temperature | Open |
| R3 | **Prompt injection into the judge** — a model's answer embeds instructions to the aggregator/judge | Low | Medium | Judge/aggregator instructions live in the system prompt; constrain judge output (tiny max_tokens, validate it's a number/CORRECT-INCORRECT); deterministic graders for all closed tasks so a poisoned answer cannot flip a closed grade (Greshake et al. 2023, threat model only) | Open |
| R4 | **Cost runaway** — ensemble/cascade/benchmark POCs make many calls and overspend | Medium | High | Per-run USD budget guard (default $0.10/run) with pre-call hard stop (L5 + capstone); on-disk cache so overlapping (model,item) pairs pay once (critical for X5); suites ≤ ~12 items; debate/AutoMix cap rounds+items; reasoning models used sparingly | Open |
| R5 | **Rate limits / 429** — bursts trip provider rate limits mid-run | Medium | Medium | Harness retries 429/5xx with backoff (1.5×(attempt+1)s); serial execution (no parallel POC workers sharing a cache); fallback chain to a working model; Tier limits recorded in `01-research/pricing-quotas-limits.md` (OpenAI T1 ~500 RPM, Anthropic Opus T1 50 RPM, xAI T0 1800 RPM) | Open |
| R6 | **Model drift** — a pinned model id is renamed/removed/deprecated mid-build | Medium | Medium | Model pool live-verified 2026-06-21; gpt-4.1-nano documented deprecated but accessible (snapshot 2025-04-14) — note in versions; if a model 404s, swap within the same tier in `config.py` and re-run affected POCs; never silently price an unknown model (`usd_for` raises) | Open |
| R7 | **OpenRouter not present** — `OPENROUTER_API_KEY` is not on disk; open models + native `/generation` cost unavailable | Confirmed | Low | OpenRouter is OPTIONAL by spec; no POC depends on it; harness has an OpenRouter backend gated behind the key; if the key appears it is a pure enhancement (open models + native cost), never a blocker; recorded in `04-logs/access-blockers.md` | Active (accepted) |
| R8 | **Routing collapse** — a trained router degenerates to route ~all queries to one model (= a baseline) due to imbalanced labels / no regularization | Medium | Medium | Assert `0 < pct_cheap < 1` for interior thresholds (L2b/X5 regression); ensure the labeled set has both classes (some items where cheap suffices, some not); report any collapsed router honestly rather than hiding it (EquiRouter framing, research-supported) | Open |
| R9 | **Silent misroute (FM-1)** — cheap model returns 200 OK but is wrong; no error signal | High | High | Per-item correctness logged (not just success); verifier gate (L3a/X4) catches low-confidence answers and escalates; L5 explicitly demonstrates a caught misroute; self-verification miscalibration documented (X4) as why a meta-verifier exists | Open |
| R10 | **Self-verification miscalibration** — cheap model over-trusts its own wrong answer, so the cascade does not escalate when it should (X4) | Medium | Medium | X4 sweeps the confidence threshold and documents ≥1 case where self-verify was confident but wrong; reported as the reason AutoMix's POMDP meta-verifier (cited, not reproduced) exists; honest about whether escalation actually saved cost | Open |
| R11 | **xAI ticks→USD ambiguity** — harness uses `1e-9`, xAI docs state `1e-10` (10^10 ticks/USD); native cost could be off 10× | Medium | Low | L0 captures a real grok-4.3 `cost_in_usd_ticks` and reconciles against the uniform `usd_for` estimate; record the correct factor in `surprises.md`/versions; native cost is recorded for transparency only — the uniform per-token cost (not ticks) drives all comparisons, so this does not corrupt the Pareto numbers | Open |
| R12 | **Reasoning-model empty answers** — gpt-5/o-series/grok burn the budget on hidden reasoning and return "" at small max_tokens | Medium | Medium | Harness floors reasoning models to `REASONING_FLOOR=2048` so they actually answer; this makes them legitimately more expensive (realistic); assert non-empty `text` in L0 smoke for each provider | Open |
| R13 | **Cheap-ensemble assumption bias** — assuming cheap-together beats single SOTA (it may not on our suite) | Medium | Medium | X1/X2/X3 RED tests do NOT assert cheap wins; the headline is the REAL measured outcome (win OR loss); regression tests force the honest verdict to be recorded; loss is a valid, reportable result | Open |
| R14 | **Sandbox socket binding** — L4 local HTTP server cannot bind a port in the workspace sandbox | Low | Medium | Use a loopback port + in-process thread; if binding is blocked, record the limitation in `surprises.md` and fall back to the in-process adapter (L3c) for the gateway-shape proof while keeping the ledger; never mock the model boundary | Open |
| R15 | **Subprocess code execution blocked** — `code_grader` can't run a subprocess in the sandbox | Low | High | Coding grader runs a short-timeout subprocess; if blocked, record in `04-logs/access-blockers.md` and fall back to math/qa suites for the routing demos while flagging coding as blocked — never fake a coding pass | Open |
| R16 | **Cache staleness / poisoning** — a cached wrong/empty answer keeps a POC green across re-runs | Low | Medium | Cache key includes model+messages+system+temperature+max_tokens+nonce; clearing `.cache` re-bills but re-verifies; first-run is the billed/truth event; do not cache across a model-pool change without clearing | Open |
